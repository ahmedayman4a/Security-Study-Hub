🔐 Lecture 01: AES & Classical Cryptography Q1–Q12
1
Cryptanalysis
Which of the following BEST describes the primary goal of cryptanalysis?
aTo securely encrypt messages, preventing unauthorized access.
bTo develop new cryptographic algorithms and protocols.
cTo analyze and break cryptographic systems, revealing the underlying plaintext or key.
dTo manage and distribute cryptographic keys securely.
Explanation
✓ Correct: (c)
Cryptanalysis is the discipline of analyzing and breaking cryptographic systems to obtain plaintext or keys without authorized access. Designing systems is cryptography; key management and algorithm development are separate fields. The adversarial, analytical role of finding weaknesses is the defining characteristic of cryptanalysis.
2
Block vs Stream
What is the key difference between stream ciphers and block ciphers?
aStream ciphers encrypt data one bit or byte at a time, while block ciphers encrypt data in fixed-size blocks.
bStream ciphers are generally faster than block ciphers.
cBlock ciphers use a key stream generated from the key and an IV, while stream ciphers directly encrypt each bit with the key.
dStream ciphers are more resistant to known-plaintext attacks than block ciphers.
Explanation
✓ Correct: (a)
The fundamental structural difference is granularity: stream ciphers process one bit or byte at a time using a keystream, while block ciphers divide the entire message into fixed-size blocks (e.g., 128 bits for AES) and encrypt each block as a unit. Option (c) reverses the description — using a keystream is exactly what stream ciphers do, not block ciphers.
3
Kerckhoffs's Principle
What is the generally accepted consensus regarding the security of cryptographic algorithms?
aThe security of a cryptographic system should primarily rely on the secrecy of the algorithm itself.
bA strong cryptographic system relies on the secrecy of the key, not the algorithm.
cKeeping the algorithm secret is an effective defense against all types of attacks.
dOpen-source algorithms are inherently less secure than proprietary algorithms.
Explanation
✓ Correct: (b)
Kerckhoffs's Principle (1883): a cryptosystem should be secure even if everything except the key is public. "Security through obscurity" is unreliable because algorithms get leaked or reverse-engineered. Modern standards like AES are fully public and globally analyzed, making them stronger, not weaker.
4
Cryptanalysis Attacks
In a codebook attack, what does the attacker need to succeed?
aOnly ciphertexts without any plaintext.
bThe encryption key.
cAccess to the encryption algorithm's source code.
dA collection of known plaintext-ciphertext pairs.
Explanation
✓ Correct: (d)
A codebook attack builds a lookup table mapping plaintexts to ciphertexts — requiring known plaintext-ciphertext pairs encrypted under the same key. Once the table is built, any new ciphertext can be looked up directly without breaking the key explicitly.
5
Key Schedule
What is the role of the key schedule in a block cipher?
aTo expand the master key into multiple round keys.
bTo perform byte substitutions.
cTo add confusion to the cipher.
dTo break plaintext into fixed-size blocks.
Explanation
✓ Correct: (a) — confirmed by Questions of Doubt
The key schedule expands the master key into multiple round subkeys, one per round. Applying those keys in AddRoundKey does add confusion — but that is the role of the AddRoundKey operation, not the key schedule itself. The key schedule's sole job is expansion/derivation.
6
Block Cipher Design
What is the main function of confusion in a block cipher?
aTo shuffle bits within the block.
bTo expand the key into multiple subkeys.
cTo compress the plaintext before encryption.
dTo obscure the relationship between the key and ciphertext.
Explanation
✓ Correct: (d)
Shannon's concept of confusion makes the key-ciphertext relationship as complex as possible — achieved through substitution (like the AES S-box). Option (a) describes diffusion, which spreads statistical structure across the block. These are two distinct and complementary properties.
7
Block Cipher Design
Why do modern block ciphers use multiple rounds?
aTo increase key size.
bTo enhance confusion and diffusion.
cTo allow variable block sizes.
dTo reduce encryption speed for security.
Explanation
✓ Correct: (b)
A single round provides inadequate confusion and diffusion. Each additional round mixes the data further, compounding the effects. After enough rounds (e.g., 10 for AES-128), even a single-bit input change affects approximately half the output bits — the avalanche effect — which is the security goal.
8
Vigenère Cipher
What happens if the keyword in the Vigenère Cipher is as long as the plaintext and never reused?
aIt becomes unbreakable (like a one-time pad).
bIt becomes equivalent to a Caesar cipher.
cIt turns into a transposition cipher.
dIt loses its polyalphabetic property.
Explanation
✓ Correct: (a)
A Vigenère key as long as the message, used only once, is mathematically a one-time pad (OTP) — provably information-theoretically secure. Each character is shifted by an independently random amount, so the ciphertext reveals zero information about the plaintext. The regular Vigenère is breakable only because its key repeats, creating patterns.
9
Key Length & Security
How does doubling the key length of a block cipher affect brute-force effort?
aEffort increases exponentially.
bEffort remains the same.
cEffort doubles.
dEffort squares.
Explanation
✓ Correct: (d) — confirmed by Questions of Doubt as the most accurate
If a key is \(n\) bits, brute force requires \(2^n\) operations. Doubling to \(2n\) bits requires \(2^{2n} = (2^n)^2\) operations — the effort squares. Example: 56-bit → \(2^{56}\); 112-bit → \(2^{112} = (2^{56})^2\). The Questions of Doubt document confirmed (d) is the most precise answer.
10
AES vs DES
Why is AES widely adopted over older ciphers like DES?
aLarger block size (128 vs. 64 bits) and longer keys.
bFaster software implementation.
cPublicly designed and analyzed (no backdoors).
dAll of the above.
Explanation
✓ Correct: (d) — confirmed by Questions of Doubt
All three are correct: (1) AES has a 128-bit block vs DES's 64 bits — twice the data per cycle; (2) AES uses byte-oriented SPN operations that map efficiently to modern CPUs, while DES's bit-level Feistel operations are slower in software; (3) AES was selected via an open NIST competition (1997–2001) with worldwide cryptanalysis, whereas DES's S-box design criteria were initially secret.
11
AES Operations
What is the purpose of the MixColumns step in AES?
aTo substitute bytes using a lookup table.
bTo shift rows cyclically.
cTo diffuse data across columns via matrix multiplication.
dTo XOR the round key.
Explanation
✓ Correct: (c)
MixColumns provides diffusion — changes in one byte spread to all four bytes in a column. Each column is treated as a polynomial over GF(2⁸) and multiplied by a fixed matrix. After MixColumns, each output byte depends on all four input bytes. The other operations: (a) SubBytes — S-box substitution; (b) ShiftRows — row permutation; (d) AddRoundKey — XOR with subkey.
12
AES S-box
How is the AES S-box constructed?
aBy rotating bits in each byte.
bBy shuffling bytes randomly.
cThrough a lookup table provided by NIST.
dUsing a mathematical formula involving multiplicative inverses in GF(2⁸).
Explanation
✓ Correct: (d)
The AES S-box has a precise two-step construction: (1) Compute the multiplicative inverse in GF(2⁸) — 0x00 maps to itself; (2) Apply an affine bit transformation. This design provides strong non-linearity and resistance to differential and linear cryptanalysis. Option (c) is misleading — you can implement the S-box as a lookup table, but that table was derived mathematically.
🔗 Lecture 02: Block Ciphers, Stream Ciphers & Key Distribution Q13–Q29
13
Multiple DES
What major attack makes 2DES insecure despite its longer key?
aMeet-in-the-middle attack.
bDifferential cryptanalysis.
cFrequency analysis.
dReplay attack.
Explanation
✓ Correct: (a)
The MITM attack encrypts a known plaintext P forward under all \(2^{56}\) possible K1 values, stores the results, then decrypts the corresponding ciphertext C backward under all \(2^{56}\) K2 values, looking for matches. Total cost ≈ \(2^{57}\) — barely more than single DES. 2DES's 112-bit combined key provides essentially no real security improvement over 56 bits.
14
DES as a Group
What would be the theoretical security impact if DES were a group?
aDES would become resistant to differential cryptanalysis.
bAll multiple-DES variants would collapse to single-DES security.
cKey size could be reduced to 40 bits.
dThe Feistel network would become redundant.
Explanation
✓ Correct: (b)
If DES were closed under composition, \(E_{K_2}(E_{K_1}(P)) = E_{K_3}(P)\) for some \(K_3\) in the 56-bit key space. 2DES would provide no more security than single DES. DES is mathematically confirmed NOT to be a group — which is why multi-DES has genuine value.
15
3DES EDE Mode
In 3DES, why is the middle step a decryption (EDE mode) rather than encryption?
aTo allow backward compatibility with single DES when K₁ = K₂ = K₃.
bTo confuse attackers with inverse operations.
cTo enable parallel processing.
dTo meet NIST's encryption standards.
Explanation
✓ Correct: (a)
When \(K_1 = K_2 = K_3 = K\), EDE becomes \(E_K(D_K(E_K(P)))\). The inner E produces X; \(D_K(X)\) immediately undoes it (recovering P); the outer \(E_K\) applies once — exactly single DES. This enabled compatible migration from legacy DES systems with no protocol changes.
16
Block Cipher Modes
Which block cipher mode encrypts each block independently, making identical plaintext blocks produce identical ciphertext blocks?
aCBC (Cipher Block Chaining).
bECB (Electronic Codebook).
cCTR (Counter).
dGCM (Galois/Counter Mode).
Explanation
✓ Correct: (b)
ECB applies the block cipher directly to each plaintext block with the same key — no chaining, no randomization. Identical plaintext blocks always produce identical ciphertext blocks, leaking structural information (the "ECB penguin" problem). All other modes introduce state or randomization to prevent this.
17
Block Cipher Modes
What is the main advantage of CTR mode?
aBuilt-in error propagation.
bResistance to IV reuse.
cParallelizable encryption/decryption.
dFixed padding requirements.
Explanation
✓ Correct: (c)
CTR generates keystream blocks by encrypting successive counter values. Since each block depends only on the counter (not on previous ciphertext blocks), all blocks can be computed independently and in parallel. Note: IV reuse is a critical vulnerability in CTR — the opposite of option (b).
18
Block Cipher Modes
What happens if an IV is reused in CTR mode?
aEncryption fails.
bBlocks become correlated.
cThe key is compromised.
dKeystream reuse breaks confidentiality.
Explanation
✓ Correct: (d) — confirmed by Questions of Doubt
Reusing the same nonce/IV and key generates the same keystream. XORing both ciphertexts: \(C_1 \oplus C_2 = P_1 \oplus P_2\). With language statistics, the plaintexts can be recovered — keystream reuse breaks confidentiality. The Questions of Doubt document confirmed (d) and noted that (b) is incorrect because what becomes correlated is the keystream overall, not individual blocks.
19
Block Cipher Modes
What is the critical security requirement for an IV in CBC mode?
aIt must be secret.
bIt must equal the block size.
cIt must be random.
dIt must be reused for efficiency.
Explanation
✓ Correct: (c)
The CBC IV must be random (unpredictable) — not merely unique. A predictable IV enables chosen-plaintext attacks to verify guesses about the plaintext. The IV does NOT need to be secret — it is typically transmitted in cleartext alongside the ciphertext. Reusing an IV reveals whether two messages share the same first block.

21
RC4 Stream Cipher
What happens if an RC4 key is reused?
aPlaintexts can be extracted via XOR.
bEncryption speeds up.
cThe key schedule resets automatically.
dError propagation increases.
Explanation
✓ Correct: (a)
RC4 generates a deterministic keystream from the key. With key reuse: \(C_1 \oplus C_2 = P_1 \oplus P_2\). Via "crib dragging" and language statistics, both plaintexts can be recovered. This broke WEP (802.11b) which fatally reused RC4 keys.
22
Key Distribution
What is a "session key" in Needham-Schroeder?
aA long-term public key.
bA hash of the shared secret.
cA temporary symmetric key.
dA digital certificate.
Explanation
✓ Correct: (c)
A session key is a randomly generated symmetric key created fresh for each communication session, discarded after the session ends. Unlike long-term keys (password hashes, service secrets) which persist indefinitely, session keys are ephemeral. In Needham-Schroeder, the KDC generates and distributes the session key.
23
Needham-Schroeder
In the Needham-Schroeder protocol, what is the purpose of nonces?
aTo encrypt the session key.
bTo compress data.
cTo generate digital signatures.
dTo ensure message freshness.
Explanation
✓ Correct: (d)
A nonce (number used once) is a random value included in messages. When a server echoes back a nonce, the requester knows the response is fresh and not a replay. If an attacker replays an old message, they won't have the correct nonce. Nonces are the primary defense against replay attacks in Needham-Schroeder.
24
Needham-Schroeder
Which cryptographic primitive does the original Needham-Schroeder protocol rely on?
aHash functions.
bSymmetric-key encryption.
cDigital signatures.
dElliptic curve cryptography.
Explanation
✓ Correct: (b)
The original 1978 Needham-Schroeder protocol uses exclusively symmetric-key encryption. Every party shares a long-term symmetric key with the trusted KDC, which uses master keys to encrypt session key bundles. No public-key crypto, no hashing for authentication, no digital signatures.
25
Kerberos
Which three main entities interact in Kerberos?
aClient, Server, Certificate Authority.
bUser, Router, DNS Server.
cBrowser, Web Server, Proxy.
dClient, Authentication Server, Ticket Granting Server.
Explanation
✓ Correct: (d)
Kerberos: (1) Client — the user; (2) Authentication Server (AS) — verifies identity using the user's long-term key and issues a TGT; (3) Ticket Granting Server (TGS) — uses the TGT to issue service-specific tickets. AS and TGS together form the KDC (Key Distribution Center).
26
Kerberos
Why does Kerberos use timestamps?
aTo prevent replay attacks.
bTo log user activity.
cTo synchronize clocks across networks.
dTo encrypt session keys.
Explanation
✓ Correct: (a)
Kerberos embeds timestamps in tickets. Servers check that the timestamp is within ±5 minutes. A replayed ticket from minutes earlier is rejected as expired. This is Kerberos's primary replay attack defense — which is why Kerberos requires synchronized clocks, not the other way around.
27
Kerberos
Which key is used to encrypt the Ticket Granting Ticket (TGT)?
aThe user's public key.
bA session key negotiated with AES.
cThe TGS's secret key (known only to the KDC).
dThe service's private key.
Explanation
✓ Correct: (c)
The TGT is encrypted with \(K_{TGS}\) — the TGS's long-term secret key, shared only between the TGS and the KDC. The client receives but cannot decrypt the TGT. This opaque credential is what the client presents to the TGS for service tickets — enabling single sign-on without re-contacting the AS.
28
Kerberos
Which key is NOT stored in the KDC database?
aUser's long-term key (password hash).
bClient/service session keys.
cTGS's secret key.
dService's long-term key.
Explanation
✓ Correct: (b)
Session keys are generated freshly per authentication exchange and are never stored in the KDC database. The KDC stores permanent credentials: user long-term keys, service long-term keys, and its own keys. If the KDC's database is ever compromised, session keys from past sessions cannot be recovered — because they were never stored.

aSession keys.

🔑 Lecture 03: Public-Key Cryptography & RSA Q30–Q38
30
RSA Security
What is the foundation of RSA's security?
aThe difficulty of factoring large integers.
bThe hardness of solving discrete logarithms.
cThe complexity of hash collisions.
dThe secrecy of its S-boxes.
Explanation
✓ Correct: (a)
Given \(n = p \times q\) with large primes, there is no known polynomial-time algorithm to factor \(n\). If an attacker could factor \(n\), they could compute \(\phi(n) = (p-1)(q-1)\) and find \(d = e^{-1} \bmod \phi(n)\). Discrete logarithms are the basis for Diffie-Hellman and ElGamal, not RSA.
31
RSA Key Generation
What is the correct order for RSA key generation?
aSelect \(e\) → Choose \(n\) → Factor \(n\) → Compute \(d\).
bGenerate \(d\) → Find \(e\) → Randomize \(n\) → Pick \(p, q\).
cCompute \(n\) → Solve for \(p, q\) → Derive \(e\) → Choose \(d\).
dChoose primes \(p, q\) → Compute \(n = pq\) → Select \(e\) → Calculate \(d\).
Explanation
✓ Correct: (d)
Sequence: (1) Choose large random primes \(p\) and \(q\); (2) Compute \(n = p \times q\); (3) Compute \(\phi(n) = (p-1)(q-1)\); (4) Select \(e\) with \(\gcd(e, \phi(n)) = 1\); (5) Compute \(d \equiv e^{-1} \pmod{\phi(n)}\). You cannot factor \(n\) to get back \(p\) and \(q\) — that's the whole security basis of RSA.
32
RSA Key Generation
In RSA, what condition must the public exponent \(e\) satisfy?
a\(e > n\).
b\(e\) must be prime.
c\(1 < e < \phi(n)\) and \(\gcd(e, \phi(n)) = 1\).
d\(e \equiv 1 \pmod{p}\).
Explanation
✓ Correct: (c)
Two conditions: (1) \(1 < e < \phi(n)\) — valid exponent; (2) \(\gcd(e, \phi(n)) = 1\) — ensures \(d = e^{-1} \bmod \phi(n)\) exists. \(e\) does NOT need to be prime (common misconception) — it just needs to be coprime with \(\phi(n)\). Common choices: 3, 17, 65537 (all prime but chosen for efficiency).
33
RSA Private Key
How is the private exponent \(d\) computed?
a\(d \equiv e^{-1} \pmod{\phi(n)}\).
b\(d = n\).
c\(d = e \bmod p\).
d\(d\) is randomly selected.
Explanation
✓ Correct: (a)
\(d\) is the modular multiplicative inverse of \(e\) modulo \(\phi(n)\): \(d \cdot e \equiv 1 \pmod{\phi(n)}\), computed via the Extended Euclidean Algorithm. Raising a ciphertext to the power \(d\) undoes the encryption because \(e \cdot d \equiv 1 \pmod{\phi(n)}\) and Euler's theorem gives \(m^{\phi(n)} \equiv 1 \pmod{n}\).
34
RSA Performance
Which operation is computationally expensive in RSA?
aDecryption (using \(d\)).
bEncryption (using \(e\)).
cGenerating \(p\) and \(q\).
dCalculating gcd.
Explanation
✓ Correct: (a)
Decryption computes \(m = c^d \bmod n\). Since \(d\) is ~2048 bits, square-and-multiply requires ~2048 modular squarings. Encryption with small \(e = 65537 = 2^{16}+1\) requires only 17 multiplications. This asymmetry is why RSA-CRT (which speeds up decryption ~4×) is so valuable in practice.
35
RSA Usage
Why is RSA rarely used for bulk encryption?
aLack of authentication.
bSlow performance.
cFixed block size.
dInability to encrypt large files.
Explanation
✓ Correct: (b)
RSA is ~1000× slower than AES for bulk data. In practice, RSA only encrypts a small symmetric session key (hybrid encryption), which then encrypts the actual data with AES. This pattern is used in TLS, PGP, and SSH.
36
RSA Vulnerabilities
Which vulnerability arises from reusing the same \(n\) with different \(e\)?
aNonce reuse.
bTiming attack.
cPadding oracle attack.
dCommon modulus attack.
Explanation
✓ Correct: (d)
If message \(m\) is encrypted under same \(n\) with two exponents \(e_1, e_2\) where \(\gcd(e_1,e_2)=1\): find \(a,b\) with \(a e_1 + b e_2 = 1\) (Bézout), then \(c_1^a \cdot c_2^b = m^{a e_1 + b e_2} = m \pmod{n}\). The plaintext is recovered without knowing either private key. Each RSA key pair must use its own distinct modulus.
37
RSA-CRT
What is the primary purpose of using CRT with RSA?
aTo increase the key size.
bTo enable quantum resistance.
cTo replace modular exponentiation.
dTo speed up decryption/signature generation by ~4×.
Explanation
✓ Correct: (d)
RSA-CRT computes \(m_p = c^{d_p} \bmod p\) and \(m_q = c^{d_q} \bmod q\) separately, then combines via CRT. Since \(p\) and \(q\) are each ~half the size of \(n\), and modular exponentiation cost scales cubically with number size, the speedup is approximately \(2^3/2 = 4\times\). The results are combined: \(m = m_p \cdot q \cdot (q^{-1} \bmod p) + m_q \cdot p \cdot (p^{-1} \bmod q) \pmod{n}\).
38
RSA-CRT
Which step is NOT part of RSA-CRT decryption?
aCompute \(m_p = c^{d_p} \bmod p\) and \(m_q = c^{d_q} \bmod q\).
bFactor \(n\) into \(p\) and \(q\) dynamically.
cRecombine results via CRT.
dVerify the result modulo \(n\).
Explanation
✓ Correct: (b)
RSA-CRT does NOT factor \(n\) dynamically. \(p\) and \(q\) are computed once at key generation and stored as part of the private key. Dynamically factoring \(n\) during decryption would be as hard as breaking RSA entirely — defeating the purpose. The CRT speedup comes from having \(p\), \(q\), \(d_p\), \(d_q\), and \(q^{-1} \bmod p\) pre-computed and stored in the private key.
📜 Lecture 04: More Public-Key Cryptography Q39–Q48
39
PKI / MITM
What is one of the most common ways to prevent MITM attacks?
aUsing strong passwords.
bUsing a VPN without encryption.
cImplementing encryption and digital certificates.
dDisabling firewalls.
Explanation
✓ Correct: (c)
MITM attacks succeed when an attacker can intercept and substitute keys. Defense requires: (1) Encryption — attackers can't read intercepted traffic; (2) Digital certificates — bind public keys to verified identities, so Alice can verify a key genuinely belongs to Bob, not a MITM impostor. Strong passwords help with authentication but don't prevent key substitution.
40
PKI / Certificates
What is the purpose of a Certificate Revocation List (CRL)?
aTo store all active certificates.
bTo generate new digital certificates.
cTo list certificates that are no longer valid.
dTo provide encryption keys for secure communication.
Explanation
✓ Correct: (c)
A CRL is a signed list of certificate serial numbers that have been revoked before their expiration date (e.g., due to key compromise or identity changes). Clients check the CRL before trusting a certificate. The modern alternative is OCSP (Online Certificate Status Protocol) for real-time revocation checks.
41
PKI / Certificates
What is a root certificate in the context of a Certificate Authority?
aA certificate that is self-signed and serves as the trust anchor for other certificates.
bA certificate stored on the client device and used to encrypt traffic.
cA certificate that encrypts data between the CA and the end-user.
dA temporary certificate used for secure email.
Explanation
✓ Correct: (a)
A root CA certificate is self-signed (it signs itself) and is the ultimate trust anchor in the PKI hierarchy. It's pre-installed in operating systems and browsers. The chain of trust: end-entity cert → Intermediate CA cert → Root CA cert. Trust in the root is bootstrapped by the OS/browser vendor's decision to include it in their trusted root store.
42
PKI / Certificates
What does a digital certificate issued by a CA typically include?
aOnly the private key of the certificate holder.
bA hashed password and the certificate holder's contact information.
cA list of trusted websites and their encryption standards.
dThe public key, the identity of the certificate holder, and the CA's signature.
Explanation
✓ Correct: (d)
A certificate (X.509) binds a public key to an identity: (1) subject's public key; (2) identity info (domain, organization, country); (3) CA's digital signature — signed with the CA's private key, verifiable by anyone with the CA's public key; (4) validity period. Certificates never contain private keys — those stay on the owner's system.
43
PKI / RA
What is the role of a Registration Authority (RA) in PKI?
aTo revoke compromised certificates.
bTo verify the identity of users before certificate issuance.
cTo store private keys securely.
dTo encrypt data transmissions.
Explanation
✓ Correct: (b)
The RA handles identity verification on behalf of the CA — checking documents, domain ownership, or business registration — then forwards verified requests to the CA for signing. RAs offload the identity-checking burden from the CA. RAs do NOT sign certificates; only CAs do.
44
Diffie-Hellman
Which property does Diffie-Hellman provide?
aKey exchange.
bNon-repudiation.
cData integrity.
dHashing.
Explanation
✓ Correct: (a) — confirmed by Questions of Doubt
DH provides key exchange — two parties establish a shared secret over a public channel. It does NOT provide: authentication (vulnerable to MITM without it), non-repudiation (DH doesn't sign anything), data integrity, or hashing. To add authentication, DH is combined with digital signatures or certificates (as in DHE within TLS).
45
DH Security
Why can't an attacker easily derive the shared secret from public values in DH?
aDue to the hardness of the discrete logarithm problem.
bBecause the keys are encrypted.
cLack of computational power.
dUse of quantum-resistant algorithms.
Explanation
✓ Correct: (a)
Public values are \(g^a \bmod p\) and \(g^b \bmod p\). The shared secret is \(g^{ab} \bmod p\). An eavesdropper who sees \(g^a \bmod p\) must find \(a\) — the discrete logarithm problem. No known polynomial-time classical algorithm can solve this for large \(p\). Standard DH is not quantum-resistant — Shor's algorithm breaks it in polynomial time.
46
Diffie-Hellman
What is the main advantage of Diffie-Hellman Ephemeral (DHE)?
aFaster key generation.
bSmaller key sizes.
cForward secrecy.
dBuilt-in authentication.
Explanation
✓ Correct: (c)
DHE generates fresh DH parameters for every session. Compromising the server's long-term key later cannot decrypt past sessions — Perfect Forward Secrecy (PFS). Static DH (with fixed parameters) lacks this: compromise the long-term key, decrypt all past sessions. DHE is slightly slower due to per-session key generation but this is the price of PFS.
47
ElGamal
What type of cryptographic system does the ElGamal digital signature scheme belong to?
aSymmetric encryption.
bHashing algorithm.
cQuantum cryptography.
dAsymmetric encryption.
Explanation
✓ Correct: (d)
ElGamal is an asymmetric (public-key) system based on the discrete logarithm problem, using a private key \(a\) and public key \(\beta = g^a \bmod p\). The private key signs; the public key verifies. Developed by Taher Elgamal in 1985, it's the basis for DSA (NIST's Digital Signature Algorithm).
48
ElGamal
What is the main purpose of the ElGamal digital signature?
aEncrypt messages for confidentiality.
bVerify the authenticity and integrity of a message.
cGenerate symmetric keys.
dSpeed up network communication.
Explanation
✓ Correct: (b)
The ElGamal digital signature verifies: (1) Authenticity — the message was signed by the holder of the corresponding private key; (2) Integrity — the message hasn't been modified. Signing does NOT encrypt the message. Critical: the nonce \(K\) used in ElGamal signing must be unique per message — reusing \(K\) allows the private key to be recovered (this broke Sony's PS3).

Hash Functions
What is the primary purpose of a cryptographic hash function?
aGenerate a fixed-size output from variable-size input.
bEncrypt data for secure transmission.
cProvide compression for large files.
dCreate digital signatures.
Explanation
✓ Correct: (a)
The defining characteristic of a hash function is mapping any input of arbitrary length to a fixed-size output. A 1-byte file and a 10GB file both produce the same size digest (e.g., 256 bits for SHA-256). Hash functions are not encryption (no key, not reversible) and not compression (the output is much smaller than the input and irreversible).
50
Hash Properties
What does "collision resistance" mean in hash functions?
aIt should be hard to find two different inputs with the same hash.
bIt should be easy to reverse the hash output.
cThe hash output should always be unique.
dThe hash function should use a secret key.
Explanation
✓ Correct: (a)
Collision resistance: finding any \(m_1 \neq m_2\) with \(H(m_1) = H(m_2)\) must be computationally infeasible. Hash outputs are NOT guaranteed unique — by the pigeonhole principle, collisions must exist (infinite inputs, finite outputs). Collision resistance only requires finding them be hard. Option (d) describes a MAC, not a plain hash.
51
Birthday Attack
Which attack involves finding any two inputs that produce the same hash?
aPre-image attack.
bBirthday attack.
cBrute-force attack.
dMan-in-the-middle attack.
Explanation
✓ Correct: (b)
The birthday attack exploits the birthday paradox: for an \(n\)-bit hash, ~\(2^{n/2}\) random inputs gives ~50% probability of finding any two with the same hash — far fewer than the full \(2^n\) brute force space. A pre-image attack is different: it starts with a specific hash \(h\) and tries to find any \(m\) with \(H(m) = h\), costing \(2^n\) work.

53
Hash Properties
What is the difference between weak (second pre-image) and strong collision resistance?
aWeak collision resistance deals with fixed inputs, while strong collision resistance deals with any two inputs.
bWeak collision resistance is about reversing the hash, while strong is about finding any collision.
cWeak collision resistance requires a secret key, while strong does not.
dThere is no difference; they mean the same thing.
Explanation
✓ Correct: (a)
Weak (second pre-image): given a specific, fixed \(m_1\), find any \(m_2 \neq m_1\) with \(H(m_1) = H(m_2)\). The attacker is constrained to a target.
Strong collision resistance: find any two different messages with the same hash — no fixed constraint. Strong is harder to achieve; the birthday attack exploits the gap between the two (\(2^{n/2}\) vs \(2^n\) work).
54
Hash Properties
If a hash function is strong collision-resistant, does it imply weak collision resistance?
aNo, because they are independent properties.
bOnly if the hash function is also pre-image resistant.
cYes, because strong collision resistance is a stronger property.
dOnly for keyed hash functions.
Explanation
✓ Correct: (c) — confirmed by Questions of Doubt
If it is hard to find ANY two inputs with the same hash (strong), then it is certainly hard to find a second input that collides with a specific fixed input (weak) — the weak problem is a special case. Contrapositive: if you could break weak collision resistance (find a collision with a given target), you'd have a collision — breaking strong too. So strong → weak holds.

Hash Construction
What is the most common structural design used in traditional hash functions like SHA-1 and SHA-2?
aSponge construction.
bMerkle-Damgård construction.
cFeistel network.
dDavies-Meyer construction.
Explanation
✓ Correct: (b)
Merkle-Damgård (1979) underlies MD5, SHA-1, and the SHA-2 family. The message is padded, then processed iteratively: compression function \(f\) takes the current state \(H_{i-1}\) and message block \(M_i\) to produce \(H_i\), starting from a fixed IV. The sponge construction (a) is used in SHA-3/Keccak. Davies-Meyer is one specific way to construct a compression function from a block cipher.
56
Hash Construction
Which component of a hash function processes one block at a time and updates the internal state?
aInitialization Vector (IV).
bPermutation function.
cKey schedule.
dCompression function.
Explanation
✓ Correct: (d)
The compression function is the iterative core of Merkle-Damgård. It takes the current chaining value \(H_{i-1}\) and message block \(M_i\), outputting the next state \(H_i\). "Compression" means combining two inputs (total size \(> n\) bits) into one \(n\)-bit output. The IV provides the initial state; it does not process blocks.
57
Hash Construction
What is the role of the "initialization vector (IV)" in a hash function?
aActs as a secret key for encryption.
bProvides the initial fixed value for the hash computation.
cEnsures collision resistance.
dShortens the output hash.
Explanation
✓ Correct: (b)
The IV in a hash function is a standardized, publicly-known constant that serves as the initial chaining value \(H_0\) for the first compression function call. It is NOT a secret (unlike a CBC IV which must be unpredictable). SHA-256's IV consists of specific 32-bit words derived from the fractional parts of square roots of the first eight primes.
58
HMAC
Which of the following components does HMAC require?
aA public key and a block cipher.
bAn IV (Initialization Vector).
cA secret key and a hash function.
dA digital certificate.
Explanation
✓ Correct: (c)
HMAC requires exactly: (1) a shared secret key \(K\) and (2) a hash function \(H\). No block cipher, IV, certificate, or asymmetric keys needed. Formula: \(\text{HMAC}(K,m) = H(K \oplus \text{opad} \| H(K \oplus \text{ipad} \| m))\). Because HMAC uses a symmetric key, it provides authentication but NOT non-repudiation.
59
HMAC vs Signatures
How does HMAC differ from a digital signature?
aHMAC uses symmetric keys; signatures use asymmetric keys.
bHMAC is faster but provides non-repudiation.
cSignatures are shorter than HMAC outputs.
dHMAC requires a certificate authority.
Explanation
✓ Correct: (a)
Fundamental difference: HMAC uses a shared symmetric key (both parties must know it); digital signatures use asymmetric keys (private to sign, public to verify). This means HMAC cannot provide non-repudiation — anyone holding the shared key could have generated the HMAC. Option (b) is wrong: HMAC does NOT provide non-repudiation.
60
HMAC Formula
What is the formula for HMAC?
a\(\text{HMAC}(K, m) = H(K \| m)\)
b\(\text{HMAC}(K, m) = H(K \oplus \text{opad} \| H(K \oplus \text{ipad} \| m))\)
c\(\text{HMAC}(K, m) = H(m \| K)\)
d\(\text{HMAC}(K, m) = \text{Encrypt}(K, H(m))\)
Explanation
✓ Correct: (b)
The inner hash \(H(K \oplus \text{ipad} \| m)\) processes the message with the inner-padded key. The outer hash wraps it: \(H(K \oplus \text{opad} \| \text{inner})\). The two different padding constants (ipad = 0x36…36, opad = 0x5C…5C) ensure inner and outer keys differ, preventing structural attacks. Options (a) and (c) are simple prepend/append constructions vulnerable to length-extension attacks.
